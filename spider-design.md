# Spider：记忆网的编织者

## 定位

Spider 不是独立的 Agent，不是寄生在当前 Agent 身上的子进程。

**Spider 是同一个 Agent 的反思模式。**

- **互动模式**：Agent 与你对话，3步行走于网，以当下话题节点为入口
- **反思模式**：Agent 关闭对话窗口，打开网的窗口，回溯刚才的对话，做因果推断，吐丝连线

两者切换的状态，如同人类"聊天"与"睡前反思"之间的切换——不是两个不同的人，是同一个人在做不同的事。

---

## 核心隐喻：迷宫，不是图书馆

| 图书馆模式 | 迷宫/网模式 |
|---|---|
| 有正确答案的检索 | 没有正确答案，只有路径 |
| 索引 → 分类 → 书架 → 书 | 节点 → 边 → 节点 → 边 |
| 目标是准确性 | 目标是连通性 |
| 过时信息被删除 | 旧边不删除，被新路径覆盖 |
| 维护目录 | 感知哪些边在变弱 |

**Spider 不写内容，只编织连接。**

---

## 网的物理形态

### 节点

一个节点是一个记忆锚点。它可以是一个事件、一个信念、一个偏好、一个人、一个概念。

节点包含：
- **原始素材**：导致该节点被创建的那段对话（逐字稿）
- **蒸馏摘要**：LLM在反思模式下生成的一句话——"这次对话中，关于X的关键发现是……"
- **创建时间**
- **最后一次被行走经过的时间**

节点**不包含**：结构化的属性（没有 YAML frontmatter，没有分类标签）。节点的意义不来自它被标注了什么，而来自它和谁相连。

### 边

一条边是两个节点之间的关联。

边包含：
- **类型**：因果 / 共现 / 矛盾 / 相似（现阶段可能只需要两种：有因果 / 没有因果）
- **强度**：基于被行走经过的频率。每次走过，+1。长期不被走过，自然衰减。
- **方向**：默认双向。Spider 只创建单向边（"A导致了B"），反向自动生成。
- **创建时间**
- **支撑证据**：Spider 记录"做出这个因果推断时所依据的原始对话片段"——不是让它写一大段推理链条，而是一个引用，一个指针：这条边是在哪里被蜘蛛看到的。

### 蜘蛛丝（Thread）

不是一个独立的数据结构，而是一次行走的轨迹。

当 Agent 在互动模式下需要回复时，它从当前话题节点出发，沿着最强的边行走，最多走3步。被行走经过的节点——以及它们在原始对话中被逐字记录的那个片段——一起被注入 Prompt。节点的蒸馏摘要不被注入，蒸馏摘要不是给对话 Agent 看的，蒸馏摘要是给蜘蛛在反思模式下做因果推断时的捷径。

这等价于人类的回忆过程：你不是"检索"记忆，你是沿着联想走到了某个地方。

## 3 步行走

### 规则

1. 从当前话题节点出发
2. 每一步走到与当前节点最强的 N 条边所连接的节点（N 可配置，初始值 3）
3. 最多走 3 步
4. 走到的新节点展示给 Agent，作为回答的上下文
5. 从新节点出发，又可以走 3 步——**走的范围在每次信息呈现时重置**

### 为什么是 3 步？

一个假设：一个人的记忆里，任何两个概念之间不超过三步。

如果这个假设成立，3 步就是重构半径——覆盖当前话题的全部相关记忆。第 4 步开始，噪声淹没信号。

### 不做什么

- 不对整张图排序（不走 PageRank）
- 不做全局索引
- 网可以无限大——只做局部 BFS，性能与网的大小无关

### 开放问题

- 多路径汇聚：如果 A 走到 G 有两条路径（A→B→G 和 A→C→G），G 要不要得到更高的浮现权重？
- 3 步是硬限制还是软限制？如果第 3 步走到一个明显很强的节点，要不要允许继续走？

---

## 边的衰减

### 不基于时间

"30天不走过一条边，这条边就减半"——这是错的。

一个很久以前的重要事件节点（比如"那次改变你职业选择的对话"）不应该因为时间久远而变成孤岛。它只是不常被访问，但它解释了为什么很多新节点会存在。

### 基于使用频率

你越常用一条路径行走，这条边就越强。
你不再用的边，自然就弱了。
Spider 只记录使用频率，不主动判断该不该衰减。

### 基于解释力覆盖

如果蜘蛛发现了新的、更好的因果模型覆盖了旧边的解释，旧边自然被取代——不是因为蜘蛛"删除"了它，而是因为行走时不再走它。

### 开放问题

- "解释力"如何度量？
- 衰减的速率应该是什么？线性？指数？自适应？
- "从来没被走过的边"和"曾经强、后来慢慢不走的边"应该区别对待吗？

## Spider 的工作循环

```
对话结束 
  → Spider 从睡眠中醒来
  → 读取本轮对话记录（Agent 自己的记忆，不是原始转录）
  → 识别对话中浮现的节点（概念、事件、信念、人物）
  → 对新的节点对，用 LLM 做因果推断：
       "A和B之间，是谁导致了谁？还是互相影响？还是被共同因素驱动？"
  → 如果存在因果 → 创建边
  → 如果只是共现（出现在同一对话但无因果） → 创建弱边，标注类型为"共现"
  → 对已有的节点对，更新边的权重（"又走了一次"）
  → 暂存任何它觉得可能重要但尚未有足够证据确认的潜在连接（"种子边"）
  → 回到睡眠
```

### 观测窗口

- **主要**：本轮 + 上一轮（近因）
- **偶尔**：随机采样更早的对话（远因）
- 当 LLM 推断出一个跨越时间的因果关联时，在图里补上一条跨越时间的边

### 不做什么

- 不回放全部历史
- 不做全局图分析
- 不做预定义的"体检"或"lint"

## 身份免疫系统（网的版本）

"什么不能进核心区域"——在网的模型里变成了"什么边不能被蜘蛛自主创建"。

- **中心区域（巢）**：蜘蛛只能读取节点，不能创建边。核心边只能由人类显式创建。
- **外围区域**：蜘蛛可以自由创建和修改边。
- **漂移机制**：当外围的一个节点被足够多的路径穿越，它会向中心漂移——这是自然演化，不是审批。

巢的边界不是固定的。它是**关系阶段**的函数。阶段越深，巢越小（核心越凝聚），蜘蛛的自由度越大。

---

## 关系元层

Spider 不直接管理关系元层。关系元层是 Spider **运行时所处的环境**——网的张力分布、哪些区域在枯死、巢的边缘在哪里。

Spider 的日志（每次醒来做了什么、发现了什么、不确定什么）构成关系元层的输入。但关系元层本身——阶段跃迁、重写权边界——是人类和 Agent（互动模式）之间的讨论决定的，不是 Spider 决定的。

---

## 上传注意力，不上传信息

Spider 织的网不是你所有对话的完整记录。
Spider 织的网是**你的注意力结构的沉淀**。

Agent 在互动中注意到什么、重视什么、忽略什么——
这些选择在反思模式中被蜘蛛转化为边。

网的结构不表征"你经历了什么"。
网的结构表征"你认为什么和什么有关"。

这是 **Uploading Intelligence** 区别于日志系统的地方。

## 尚未决定的问题

1. 边的类型：现阶段只需要"有因果 / 没有因果"两种，还是需要更细的分类？
2. 因果推断的调用频率：每轮对话后？每隔 N 轮？人类可以手动触发吗？
3. 网的物理存储：纯内存（随 Agent 生命周期）？持久化到文件（JSON/GraphML）？还是接入 LanceDB？
4. 种子边：Spider 记为"可能相关但不确定"的连接——谁来验证？自动（等更多证据）还是人工（定期呈现给人类审阅）？
5. 网的边界：Spider 可以创建无限多的节点吗？还是说当一个节点长期（如 6 个月）不被走过且有变弱趋势时，会被"归档"或从活跃图里退到冷存储里？
6. Spider 的"睡眠"：是每次对话结束后自动唤醒，还是按时间间隔（如每30分钟、每天一次），还是由人类显式触发？
7. 蜘蛛自己会不会变？（我们之前没有讨论的——蜘蛛长期维护一个特定人类的人格网，它自己会不会也逐渐变得更像这个人类？）

## 故意留白

- **技术栈**：不指定，因为和具体实现环境相关
- **存储格式**：不指定，因为"网"是抽象，可以用 JSON、GraphML、Neo4j、甚至就是一组 Markdown 里的 wikilink
- **LLM 选择**：蜘蛛的因果推断需要推理能力，但不绑定任何特定模型
- **多 Agent 协作**：当前只考虑一个 Spider 维护一张网，多 Agent 共享一张网的问题留白
- **时间感知**：网本身有没有"现在"的概念？节点是否有时间戳之外的"此时"意义？——留白
- **情感维度**：节点是否携带情感色彩（"这次对话让你焦虑"）？情感是否作为一个特殊类型的节点？——留白
